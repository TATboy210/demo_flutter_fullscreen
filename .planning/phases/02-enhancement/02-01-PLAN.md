    **4. 构建日志面板 UI (per LOG-03, LOG-04):**

    ```dart
    Widget _buildLogPanel() {
      return Card(
        margin: const EdgeInsets.all(16),
        child: Column(
          children: [
            // 标题行 + 清空按钮
            Padding(
              padding: const EdgeInsets.all(16),
              child: Row(
                children: [
                  const Text('调用日志', style: TextStyle(fontSize: 18, fontWeight: FontWeight.bold)),
                  const Spacer(),
                  IconButton(
                    icon: const Icon(Icons.delete_sweep),
                    tooltip: '清空日志',
                    onPressed: _logs.isEmpty ? null : _clearLogs,
                  ),
                ],
              ),
            ),
            const Divider(height: 1),
            // 日志列表
            SizedBox(
              height: 300,
              child: _logs.isEmpty
                  ? const Center(child: Text('暂无调用记录', style: TextStyle(color: Colors.grey)))
                  : ListView.builder(
                      reverse: true,  // 最新在上
                      itemCount: _logs.length,
                      itemBuilder: (context, index) {
                        final log = _logs[index];
                        final time = '${log.timestamp.hour.toString().padLeft(2, '0')}:'
                            '${log.timestamp.minute.toString().padLeft(2, '0')}:'
                            '${log.timestamp.second.toString().padLeft(2, '0')}';
                        return ListTile(
                          dense: true,
                          leading: Icon(
                            log.success ? Icons.check_circle : Icons.error,
                            color: log.success ? Colors.green : Colors.red,
                            size: 20,
                          ),
                          title: Text(
                            '[$time] ${log.packageName}.${log.methodName} -> '
                            '${log.success ? "成功" : "失败"}'
                            '${log.errorMessage != null ? " (${log.errorMessage})" : ""}',
                            style: const TextStyle(fontSize: 13, fontFamily: 'monospace'),
                          ),
                        );
                      },
                    ),
            ),
          ],
        ),
      );
    }
    ```

    **关键实现细节：**
    - ListView 的 reverse: true 配合 insert(0, entry) 确保最新日志在视觉顶部
    - 清空按钮在 _logs 为空时禁用（onPressed: null）
    - 日志列表使用 monospace 字体对齐时间戳
    - ListTile 的 dense: true 节省垂直空间

  <files>demo_fullscreen/lib/main.dart</files>
  <behavior>
    - _toggleFwFullscreen 成功时调用 _addLog('fullscreen_window', 'setFullScreen', true)
    - _toggleFwFullscreen 失败时调用 _addLog('fullscreen_window', 'setFullScreen', false, e.toString())
    - _toggleFfFullscreen 调用时立即记录方法调用（success 待确认）
    - flutter_fullscreen listener 的 onEnterFullScreen 回调中更新最近一条日志为成功
    - flutter_fullscreen listener 的 onLeaveFullScreen 回调中更新最近一条日志为成功
    - 如果 flutter_fullscreen 调用后长时间无 listener 回调，日志保持待确认状态
  </behavior>
  <action>
    **1. 修改 _toggleFwFullscreen 添加日志 (per LOG-01, LOG-02):**

    在 try 块中 `await FullScreenWindow.setFullScreen(_fwFullscreen)` 成功后添加：
    ```dart
    _addLog('fullscreen_window', 'setFullScreen', true);
    ```

    在 catch 块中添加：
    ```dart
    _addLog('fullscreen_window', 'setFullScreen', false, e.toString());
    ```

    **2. 修改 _toggleFfFullscreen 添加日志 (per LOG-01, LOG-02):**

    在 `FullScreen.setFullScreen(!_ffFullscreen)` 调用前添加：
    ```dart
    _addLog('flutter_fullscreen', 'setFullScreen', true);
    ```

    注意：flutter_fullscreen 的 setFullScreen 返回 void，无法直接捕获异常。
    如果需要记录失败，可以在 listener 回调中处理。但根据当前架构，listener 只在成功时触发，
    所以先记录为成功，如果后续发现异常再调整。

    **3. 保持现有防抖和错误处理逻辑不变：**

    - _fwToggling / _ffToggling 防抖锁保持不变
    - try-catch 回滚逻辑保持不变
    - SnackBar 错误提示保持不变
    - listener 模式保持不变

    **关键实现细节：**
    - 日志记录是附加操作，不影响现有全屏切换逻辑
    - fullscreen_window 的日志在 try-catch 内，能准确反映成功/失败
    - flutter_fullscreen 的日志在调用时记录，因为 void 返回无法捕获异常
    - 日志时间戳由 DateTime.now() 自动生成
